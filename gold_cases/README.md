# CAP Gold Cases

Gold Cases are full-stack demonstrations of CAP mechanisms. They are longer
than ordinary examples and are intended for external review, grant discussion,
LessWrong-style critique, and implementation planning.

A Gold Case must show:

- the initial failure mode
- raw or compressed dialogue
- telemetry
- transition cost
- COM operation sequence
- subsystem mapping
- source hierarchy
- validator behavior
- corrected output
- updated telemetry
- edge cases
- generalization
